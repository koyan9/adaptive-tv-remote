# 下一阶段开发计划

## 目标

把 `Adaptive TV Remote` 从“可运行演示原型”推进到“首个可验证家庭场景 MVP”。

## Stage 1 - 真实协议接入

### 1.1 Samsung / LG / Sony LAN 真正接入
- 用真实 HTTP / WebSocket 请求替换当前 skeleton
- 增加认证、配对、重试、超时和失败分类
- 为每个品牌补最小可用命令集：`POWER`、`VOLUME`、`HOME`、`BACK`、`DPAD`

### 1.2 网关真实 API 接入
- 将 `generic-ir` 与 `generic-hdmi-cec` skeleton 替换为真实家庭控制盒 API
- 定义网关注册、鉴权和设备绑定流程
- 增加网关在线状态、心跳和错误回退

## Stage 2 - 设备与家庭模型持久化

### 2.1 数据持久化
- 引入数据库保存家庭、房间、设备、品牌画像、配对记录和执行历史
- 用持久化设备表替换当前内存种子设备

### 2.2 配对与发现流程
- 为同网电视增加发现与确认流程
- 为网关设备增加绑定与能力校验流程
- 增加失败诊断：同网失败、认证失败、协议不支持、网关离线

## Stage 3 - 面向用户的安装体验

### 3.1 PWA 强化
- 增加首页引导、房间切换、设备收藏和最近使用记录
- 优化弱网和离线提示

### 3.2 移动端壳应用
- 将 PWA 封装为 Android / iPhone 壳应用
- 增加推送能力、局域网权限说明和后台唤醒策略

## Stage 4 - 质量与可运维能力

### 4.1 兼容性矩阵
- 建立品牌 / 型号 / 路径 / 命令支持矩阵
- 标记实验性支持与已验证支持

### 4.2 可观测性
- 增加结构化日志、命令耗时、失败原因和设备在线率指标
- 输出适合集成到控制台或管理后台的数据模型

## 建议近期优先级

### P1
- 完成一个真实 LAN 品牌的端到端接入
- 完成一个真实 IR 网关 API 的端到端接入

### P2
- 完成设备持久化与配对记录
- 完成移动端壳应用的最小封装

### P3
- 扩大品牌覆盖和兼容性矩阵
- 增加管理后台或诊断页面
