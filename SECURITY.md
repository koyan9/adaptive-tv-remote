# Security Policy

## Reporting a vulnerability

- Do not post suspected security issues in public issues.
- When this project is published as a standalone repository, use the repository security reporting channel if available.
- Until then, treat this project as pre-release software and avoid using real production secrets, real device tokens, or real personal data in commits, examples, or tests.

## Current status

- The current implementation contains mock integrations and real-protocol skeletons for demonstration only.
- Authentication values in configuration are placeholders and must be replaced with secure secret handling before production use.
